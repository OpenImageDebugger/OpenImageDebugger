# OpenImageDebugger Build System

This project uses a modern C++ build system with the following components:

## Features

- **C++20** - Modern C++ standard for better performance and features
- **Protobuf** - For efficient binary serialization of buffer data
- **Conan** - Package manager for C++ dependencies
- **Ninja** - Fast build system
- **CCCache** - Compilation caching for faster rebuilds
- **Default compiler** - Uses the system's default compiler for the target platform

## Prerequisites

Make sure you have the following tools installed:

```bash
# Install conan
pip install conan

# Install ninja (Ubuntu/Debian)
sudo apt install ninja-build

# Install ccache (Ubuntu/Debian) 
sudo apt install ccache

# Or on macOS with Homebrew
brew install ninja ccache
```

## Quick Start

### Option 1: Using the build script (recommended)

```bash
./build.sh
```

### Option 2: Manual build

```bash
# 1. Install dependencies with conan
conan install . --output-folder=build --build=missing

# 2. Configure with CMake
cmake --preset default

# 3. Build the project
cmake --build --preset default
```

## Build Configurations

The project includes two preset configurations:

- **default** - Release build with optimizations
- **debug** - Debug build with debug symbols

To use the debug configuration:

```bash
cmake --preset debug
cmake --build --preset debug
```

## Protobuf Integration

The project includes protobuf definitions in `src/ipc/buffer.proto` for:

- `BufferData` - Main message for exchanging buffer data
- `BufferMetadata` - Information about buffer structure
- `BufferRequest/BufferResponse` - Request/response protocol

Protobuf files are automatically compiled during the build process.

## Performance Features

- **CCCache**: Automatically enabled if available, significantly speeds up rebuilds
- **Ninja**: Parallel compilation by default using all available CPU cores
- **Conan**: Manages dependencies efficiently with binary caching

## Directory Structure

```
build/                  # Build artifacts (created during build)
├── conan_toolchain.cmake  # Generated by conan
├── conandeps.cmake        # Generated by conan  
└── ...

src/ipc/               # IPC module with protobuf
├── buffer.proto       # Protobuf definitions
└── CMakeLists.txt     # CMake configuration with protobuf
```

## Troubleshooting

If you encounter issues:

1. Ensure all prerequisites are installed
2. Clear the build directory: `rm -rf build/`
3. Re-run the build process
4. Check ccache stats: `ccache -s`
